Daily workflow

Read, filter, and react to weather alerts quickly.

Use these request patterns in your app or scripts for day-to-day monitoring. Every request below needs a bearer token.

Typical checks

  1. See your personal alert queue.
  2. Inspect active weather events in your area.
  3. Use state or search endpoints for broader situational awareness.

Your alerts

GET /api/alerts/user/{userId}

Primary endpoint for a user-facing alert inbox.

Status values can include pending and processed notifications depending on your flow.

Specific alert detail

GET /api/alerts/{alertId}

Use when a user opens a single alert card in your UI for full headline/description context.

Live active feed

GET /api/weather/active

Snapshot of currently active NOAA-backed alerts.

Location lookup

GET /api/weather/location?latitude={lat}&longitude={lon}

Best for map-based local checks.

State lookup

GET /api/weather/state/{stateCode}

Useful for regional dashboards and travel alerts.

Search indexed weather data

GET /api/weather/search/location/{location}
GET /api/weather/search/event/{eventType}

Fast lookup paths backed by Elasticsearch for user-facing filtering and history views.

Real-time stream

CONNECT ws://localhost:8080/ws-alerts
SUBSCRIBE /topic/alerts/{userId}

Use WebSocket when you need push-style updates instead of polling.
