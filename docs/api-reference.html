Endpoint map

Role-aware API quick reference

Every route under /api/** requires bearer auth, except token creation.

Role key

USER Read-only access to alerts, criteria reads, and weather data.

ADMIN Includes all USER access plus criteria writes and pending alerts endpoint.

Method Path Role Purpose
POST /api/auth/token Public Issue JWT access token
POST /api/criteria ADMIN Create alert criteria
PUT /api/criteria/{criteriaId} ADMIN Update alert criteria
DELETE /api/criteria/{criteriaId} ADMIN Delete alert criteria
GET /api/criteria/user/{userId} USER Read criteria for a user
GET /api/criteria/{criteriaId} USER Read one criteria record
GET /api/alerts/user/{userId} USER Get user alert feed
GET /api/alerts/{alertId} USER Get specific alert detail
GET /api/alerts/pending ADMIN Get all pending alerts
GET /api/weather/active USER Get active weather alerts
GET /api/weather/location?latitude={lat}&longitude={lon} USER Weather alerts near coordinates
GET /api/weather/state/{stateCode} USER Weather alerts by state
GET /api/weather/search/location/{location} USER Search indexed weather by location
GET /api/weather/search/event/{eventType} USER Search indexed weather by event type

Common response codes

200 / 204: Request succeeded.

400: Invalid request payload or parameters.

401: Missing/expired/invalid token.

403: Role does not allow this endpoint.
